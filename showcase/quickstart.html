Linden is an MCP server you point at your data: Postgres, S3, internal APIs, anything. Tools and resources are auto-generated, schema-validated, and traceable. Drop in a Claude, GPT, or local agent — they all speak the same protocol.

from linden import Server, tool

mcp = Server("customer-support-tools")

# Schema-typed tools become MCP endpoints.
# Validation, retries, and audit logs come free.
@tool
def find_order(order_id: str) -> Order:
    """Look up an order in the production OMS."""
    return oms.get(order_id)

@tool
def refund(order_id: str, amount: float, reason: str):
    """Issue a refund. Requires reason for the audit log."""
    return oms.refund(order_id, amount, reason=reason)

# Done. Run it.
mcp.serve(transport="sse", port=8000)
Schema-validated I/O
Tools are typed. Arguments are validated. Bad input is rejected before it reaches your code.
Sandboxed by default
Read-only by default. Mutations require explicit grants and write to the immutable audit log.
Built-in observability
Every call traced. Latency, payload, error class, and originating agent — out of the box.
Streaming responses
Long tools yield incrementally. The agent gets a typed stream, not a 30-second hang.
Quickstart · 3 steps · 4 minutes
01

Install the server

Linden ships as a small Python wheel. No daemons, no containers required.

$ pip install linden-mcp $ linden init customer-support-tools --template=python
02

Declare your tools

Annotate Python functions with @tool or @resource. Type hints become the MCP schema; docstrings become the LLM-facing description.

@tool def list_invoices(vendor: str, limit: int = 10) -> list[Invoice]: """Return the last N invoices for a vendor.""" return db.q("select * from invoices where vendor=? limit ?", vendor, limit)
03

Connect from any client

Claude Desktop, Continue, Cursor, Cody — point them at the server URL. Local stdio works too.

{ "mcpServers": { "customer-support-tools": { "url": "https://mcp.linden.io/customer-support", "headers": { "Authorization": "Bearer $LINDEN_KEY" } } } }

Typed MCP servers in one line of Python.

Point Linden at any data source — Postgres, S3, GitHub, your internal API — and expose it as a sandboxed, schema-validated tool that Claude, Cursor, Zed and any MCP client can pick up. Drop in three lines, no boilerplate.

customer_support_server.py 
from linden import Server, tool

# 1. Spin up a server
srv = Server("customer-support")

# 2. Decorate a function. Linden infers
#    the JSON schema from your type hints.
@srv.tool("refund_order", scopes=["orders:write"])
async def refund(order_id: str,
                 amount_cents: int,
                 reason: str | None = None) -> str:
    """Issue a partial or full refund. Returns the
    refund_id."""
    return await stripe.refund(order_id, amount_cents)

# 3. That's it. Mounted at /mcp · ready for Claude.
srv.serve(host="0.0.0.0", port=7233)

why teams pick linden

The boring parts of MCP — done correctly, by default.

Schema-validated I/O

Pydantic-typed args and returns become canonical JSON-Schema tool definitions — the same shape Claude expects, automatically.

Sandboxed execution

Per-call CPU & memory caps, network scopes, secrets injection via short-lived OIDC. Subprocess crashes can't take down the server.

Built-in observability

Every call lands in an audit log with caller identity, latency, cost. OpenTelemetry traces ship to your existing pipeline.

Streaming responses

Long-running tools stream progress events back to the client. No more 10-minute hangs while a database query runs.

Resources, not just tools

Expose static documents and live datasets through the MCP resource protocol — agents can browse in addition to call.

Any transport

stdio, HTTP, SSE, WebSockets, named pipes. Same Python server, configurable per deployment — local CLI to enterprise mesh.

Linden vs. rolling your own MCP server

The wins compound — auth, sandboxing, observability and streaming each take weeks to get right.

Pre-Linden

  • ~400 lines of FastAPI/WebSocket plumbing per server.
  • Hand-write the JSON schema for every tool. It drifts.
  • Subprocess crashes take down the entire process.
  • No audit log — you'll write it in a sprint nobody asked for.
  • Streaming hand-rolled with framing bugs nobody can repro.
⚡ Linden

One line. Typed. Sandboxed.

  • One @srv.tool decorator. Schema inferred from types.
  • Sandbox per call — CPU, memory, network scopes enforced.
  • Audit log + OpenTelemetry traces enabled by default.
  • Streams progress, partial results, cancellation — all in protocol.
  • 6 transports, same code path. Local-to-prod with config only.
